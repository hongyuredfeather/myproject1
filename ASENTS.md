# 个人作品集网站 开发指令

## 1. 项目概述
基于 React + TypeScript + Vite + Tailwind CSS 开发的个人技术作品集单页网站，主打深色主题、响应式布局和流畅动画，用于展示个人技能、项目和联系方式。

## 2. 开发规范
### 2.1 组件规范
- 组件类型：全部使用**函数式组件** + React Hooks（禁止使用类组件）
- 命名规范：
  - 组件文件：大驼峰命名（如 Hero.tsx）
  - 组件内部变量：小驼峰命名（如 heroTitle）
  - 常量：全大写 + 下划线（如 PRIMARY_COLOR）
- 组件拆分：遵循「单一职责」，通用逻辑抽离为自定义Hooks（如滚动动画Hook）
- 注释要求：
  - 组件顶部：注释说明组件功能、入参类型
  - 复杂逻辑：添加行内注释（如动画触发条件、响应式断点）

### 2.2 样式规范
- 优先使用 Tailwind CSS 原子类，避免手写CSS（仅全局样式/特殊动画写在globals.css）
- 自定义样式：通过 Tailwind `@apply` 封装复用样式（如 `.skill-tag { @apply px-2 py-1 rounded-full bg-purple-800/30; }`）
- 响应式断点：严格遵循 Tailwind 内置断点（sm:640px, md:768px, lg:1024px, xl:1280px）
- 颜色规范：统一使用文档指定的深色主题色值，禁止随意新增颜色

### 2.3 代码质量
- TypeScript 类型：所有变量/函数/组件入参必须定义明确类型，禁止使用 `any`
- 错误处理：链接跳转前做空值判断，图片加载失败显示默认占位图
- 兼容性：确保在 Chrome/Firefox/Safari 主流版本正常运行，移动端适配微信/QQ浏览器

## 3. 设计要求（落地细则）
- 深色主题：
  - 页面根背景：#0a0a0a
  - 主文字：#ffffff
  - 次要文字：#9ca3af
  - 卡片背景：#171717
- 强调色：渐变色 `bg-gradient-to-r from-indigo-500 to-purple-500`（Tailwind 类名），用于：
  - 按钮背景、标签文字/边框
  - 标题下划线、导航栏激活状态
- 动画要求：
  - 滚动动画：使用 Framer Motion 的 `useInView` 实现元素进入可视区域时的淡入+上移（初始透明度0，y轴偏移20px）
  - 卡片 hover：transform: translateY(-4px) + box-shadow: 0 10px 25px -5px rgba(100, 116, 139, 0.2)
  - 导航栏：滚动超过100px时，背景变为 rgba(10,10,10,0.9)，添加1px下边框（#333）

## 4. 注意事项
- 设计原则：简洁为主，避免过度动画/装饰，重点突出内容
- 交互体验：
  - 所有可点击元素（按钮、链接、卡片）添加 hover 反馈
  - 移动端点击区域最小 48px × 48px，避免误触
- 可维护性：
  - 数据与视图分离，所有静态内容放在 `src/data/` 中，便于后续修改
  - 组件复用性：通用UI（按钮、标签、卡片）抽离为公共组件
- 部署适配：
  - 图片路径使用相对路径，避免部署后404
  - 打包时设置 base 路径（Vite配置），适配Vercel部署
